# 콜드오픈 — 제품 의도 전달 (최소판)

2026-06-07. 의제: 처음 OpsPilot을 연 30초 안에 "이게 뭐고 왜 똑똑한지"가 잡히게 한다.

## 배경

정체성은 직전 세션에 확정됐다 — OpsPilot = **도구 무관 하네스 개선 플라이휠**(Cursor + Claude). 측정 절반(정정왕복·사용량)은 Claude transcript 기반이고, 개선 절반은 Claude 룰 브릿지로 양쪽이 닫혔다. 그런데 *그 정체성이 첫 화면에 안 보였다*:

- 헤더 부제가 `사용량 · eval · HITL` — 내부자 약어라 *왜*를 안 말함.
- README 리드 단락이 정체성 재정의 *이전* 문구("Claude Code 자산이 잘 작동하는지")라 Cursor·플라이휠을 안 말함.
- 정보 다이얼로그(ⓘ)·온보딩 투어는 능동적으로 눌러야 보임 → 콜드오픈엔 안 잡힘.

## 결정

1차 독자 = **본인(매일 쓰는 도구) + 공개 시 제3자·채용자**. 두 요구(미니멀 vs 설명)가 충돌 → **최소 형태**로 합의:

- **형태**: 헤더 부제 한 줄 교체. 전용 화면·배너·모달 없음(YAGNI).
- **문구 결**: 효익 리드 — 하네스 품질 재정의(사람이 덜 끼어들어도 되는 정도)를 가장 정직하게 담음.
- **README**: 채용자의 진짜 1차 접점이므로 헤더와 한 호흡으로 리드 단락 갱신.

## 변경 (2파일)

### ① 헤더 부제 — `apps/web/src/app.tsx`
- `사용량 · eval · HITL` → `Cursor·Claude로 만든 규칙·에이전트가 사람 손을 덜 타게 — 측정하고 고친다`
- 길어지므로 제목·부제를 baseline 한 줄 → **세로 2단**(`flex flex-col`)으로. `OpsPilot` 아래 작은 회색 한 줄. 헤더 높이만 한 줄 늘어남.
- 기존 `사용량 · eval · HITL`은 제거. 기능 3축 설명은 정보 다이얼로그 overview headline에 이미 있어 중복 안 만듦.

### ② README 리드 단락 — `README.md:3`
- 현재: "Claude Code로 만든 에이전트·스킬·커맨드가 진짜 잘 작동하는지, 느낌이 아니라 점수와 트레이스로 확인하는 로컬 도구입니다."
- 새로: "Cursor랑 Claude로 만든 규칙·에이전트가 진짜 사람 손을 덜 타게 됐는지를, 느낌이 아니라 추세로 보는 로컬 도구입니다. 작업 기록에서 얼마나 손이 덜 갔는지 재고, 고칠 거리를 뽑아서, 규칙·에이전트에 도로 반영하는 한 바퀴를 돕니다."
- doc-tone-plain 준수(담백 구어체), 내부어휘(정정왕복) 풀어씀.
- **스코프 한계**: 리드 단락만. 아래 "왜 만들었나"(README 줄 5~) 평가 중심 서사는 이번 범위 밖 — 후속 여지.

## 비포함 (YAGNI)
전용 콜드오픈 화면·배너·모달·온보딩 변경 없음. 헤더+README 두 군데가 한 메시지로 일관되면 끝.
