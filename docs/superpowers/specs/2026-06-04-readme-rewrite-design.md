# README 전면 재작성 — 설계

- 날짜: 2026-06-04
- 대상: `README.md`(전면), `docs/consumer-onboarding.md`(신규 분리), `docs/screenshots/`(신규 스크린샷)
- 종류: 문서. 1순위 독자 = **포트폴리오/평가자**(채용·동료가 "무엇이고 왜 잘 만들었나"를 빨리 파악).

## 배경

제품이 진화했는데 README가 옛 상태다: "5분 시작"이 **피드백·실행/트레이스·프로젝트 분리 탭**을 설명하나, 실제는 **3탭(개요·프로젝트·작업)** — 피드백+실행/트레이스가 "작업" 한 탭으로 통합됐고(드릴다운 판정→개선안), 온보딩 가이드 투어가 추가됐다. 스크린샷·평가축 설명도 옛 UI. 포트폴리오 1순위 관점으로 서사를 살리되 현 제품을 정확히 반영해 전면 재작성한다.

## 새 README 구조

1. **헤더** — 한 줄 정체성 + 히어로 스크린샷(새 작업 탭).
2. **왜 만들었나** — Harness 선언문이 남긴 빈 곳(플러그인 리뷰·버저닝·정확도 모니터링). [압축·유지]
3. **닫힌 루프** — 저작→버저닝→격리 실행→평가→채택. [유지]
4. **핵심 설계** — git=버전 단일원천 · worktree 격리 실행 · 비동기 러너 · 로컬 claude CLI. [유지]
5. **무엇을 보는가 (3탭)** — 개요 / 프로젝트 / **작업**. 작업 = 피드백+실행/트레이스 통합 드릴다운: 작업 목록(코드 작업·수동 실행) → 작업 상세(판정 → 개선안 결정 → 평가·실행 과정·검토·diff 점진 노출). ★전면 갱신(핵심).
6. **평가** — 시나리오 성공조건 자동채점 / 사람 점수·회고 / **머신 스코어러(기준-인식 자동 judge)** / LLM judge + 벤치마크(일관성)·버전 비교·트레이스·변경 diff. [갱신: 3축+ 정리, 머신 스코어러 반영]
7. **온보딩 가이드 투어** — 헤더 나침반 토글 → 첫 사용 6단계(프로젝트→스캔→작업→판정→개선안)를 스포트라이트로 안내. [신규·짧게]
8. **Cursor 피드백 루프** — Cursor·AI 작업 ingest → eval → 개선안 HITL 승인 → 자산 반영. [압축]
9. **스택 · 빠른 시작** — 모노레포 표 + 부트스트랩 한 줄 + MCP 한 줄 등록. [압축]
10. **현재 상태와 한계** — 머신 스코어러 완료 반영, 이식·플라이휠·OSS 설치 미구현은 정직히. [갱신]
