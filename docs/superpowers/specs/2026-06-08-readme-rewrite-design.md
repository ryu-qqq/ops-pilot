# README 전면 재작성

2026-06-08. 현재 README가 막연하다는 지적. mysql-bulk-insert README의 일목요연함을 본받아 다시 쓴다.

## 진단

- 첫 단락이 모호하고 겹침("손이 덜 갔는지" 반복, "한 바퀴를 돕니다"가 막연).
- "왜 만들었나"가 구어체 과잉("이거 말고 저거", "돌려보려고")으로 핵심이 풀어짐.
- "무엇을 하나"가 추상적 — 증거·구체 없음.
- 산문 일변도라 스캔할 구조가 없음.

## 본받을 것 (mysql-bulk-insert)

의사결정 스토리(문제→무엇→증거→시작), 주장 대신 측정값, 이모지·표로 계층 스캔.

## 결정

네 방향 다 채택 — 문장 또렷하게 + 구조 재편 + 시각요소(절제) + 실증.

### 구조
1. 타이틀 + 한 줄 정의 + 스택 뱃지
2. ⚡ 한눈에 — 무엇/누구/어떻게 3줄
3. 🤔 왜 만들었나 — 문제 한 호흡, 구어체 과잉 제거
4. 📦 무엇을 하나 — 측정·개선·반영 + 스크린샷
5. 📊 실제로 돌려봤나 — 도그푸드 실측 표
6. 🚀 시작하기 (유지)
7. 🧱 스택 (유지)
8. 🚧 아직 안 되는 것 (Todo)
9. 📚 더 자세히 (링크)

### 시각요소 수위
섹션 제목에 이모지 하나씩(스캔 앵커), 표는 증거·스택 둘, 뱃지는 상단 한 줄. 본문 산문은 담백 유지. 남발 안 함.

### 증거 정직성
실측(7프로젝트·자산 276·작업 146·개선안 85·반영 37=43%, 2026-05-25~06-05)을 쓰되 "효과 증명"이 아니라 "도그푸드 기록"으로. 표본 작음·교란(작업난도·숙련도) 캐비앗 명시. 부풀리지 않는다.

### 룰 처리
`.cursor/rules/doc-tone-plain.mdc`에 "README·소개 문서는 스캔용 시각요소(절제된 이모지·표·뱃지) 허용, 본문 산문은 담백 유지" 예외 추가 — 다음 편집 때 충돌 방지.
